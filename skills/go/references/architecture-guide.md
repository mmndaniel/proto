# Architecture Guide

High-level technical approach. Only what matters for a working prototype.

## What to decide

- Stack: language, framework, database
- Infrastructure: where it runs, how it's deployed
- Major components: what are the pieces and how do they connect?
- Data flow: if applicable, how does data move through the system?

## Keep it practical

Simplest stack that works. This is a prototype, not a production system.

Skip: scaling strategy, hiring implications, long-term maintenance.
