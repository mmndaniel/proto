# Architecture Guide

Decide the high-level technical approach. Focus on what matters for getting a working prototype.

## What to decide

- Stack: language, framework, database (if any)
- Infrastructure: where it runs, how it's deployed
- Major components: what are the main pieces and how do they connect?
- Processing pipeline: if applicable, how does data flow?

## Keep it practical

Pick the simplest stack that gets the job done. Optimize for speed of development. This is a prototype, not a production system.

Don't discuss: scaling strategy, hiring implications, long-term maintenance. Those come later if the prototype proves the idea.
