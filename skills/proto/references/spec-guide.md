# Spec Guide

Capture what we're building and why. This is a working document, not a formal spec.

## What to cover

- What are we building? (one paragraph)
- Why? What problem does it solve?
- Who uses it?
- What are the main user flows?
- What's in scope for the prototype?
- What's explicitly out of scope?

## Keep it short

Someone should be able to read the spec in 30 seconds and understand the goal. If it's longer than a page, cut it.
